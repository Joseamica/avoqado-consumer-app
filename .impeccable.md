# Avoqado Consumer App — Design Context

This file is the source of truth for design decisions in the Avoqado consumer mobile app (React Native / Expo). All `impeccable:*` skills and any future design work should treat this as the design brief.

## Design Context

### Users

The Avoqado consumer is a **mid-to-upper-income urban Mexican** (CDMX, GDL, MTY initially) booking **personal services and appointments** — not restaurant reservations. Real use cases, in priority order:

1. **Fitness classes** — pilates, yoga, spinning, crossfit, functional, boutique studios
2. **Spa & wellness** — massage, facial, treatments
3. **Beauty & grooming** — haircut, barber, manicure, eyelashes, aesthetics
4. **Retail-by-appointment** — jewelry showrooms, ateliers, custom fittings
5. **Healthcare** — dentist, doctor, specialists, therapy
6. **Other services** — anywhere a venue uses Avoqado for booking

The user opens the app with an intent ("I want a haircut Saturday morning near me", "Book my pilates class for tomorrow 7am"), expects to find venues quickly, see what they look like, choose a time slot, and confirm — sometimes paying a deposit via Stripe Connect.

**Context of use**: mostly mobile, often on the go (waiting room, lunch break, between meetings). Sometimes in a moment of aspiration ("treat myself this weekend"). They will also receive push notifications for confirmations and reminders.

**They authenticate** via Google Sign-In or Sign in with Apple, become a customer of one or more venues, and may carry credits/wallet balance for some venues that sell prepaid packs.

### Brand Personality

Three words: **Curada · Acogedora · Confiable** (curated · welcoming · trustworthy).

- **Curada**: Avoqado does not list every business — venues are intentional. The interface should feel like a thoughtfully edited shelf, not a yellow-pages directory.
- **Acogedora**: Warm, human, mexicana sin caricatura. The user should feel invited, not processed. Closer to entering a beautiful boutique than logging into a banking app.
- **Confiable**: For sensitive bookings (dentist, medical, aesthetic procedures), the user is handing over their body and money. The interface must feel grounded, legible, and safe — without becoming clinical or cold.

**Emotional arc** during the core flow:
1. Open app → *curiosity, gentle aspiration*
2. Search/browse → *desire to discover, low cognitive load*
3. View venue → *trust + a "this is the right place" feeling*
4. Pick slot → *confidence, no second-guessing*
5. Pay/confirm → *relief, anticipation*
6. Reminder push → *warmth, "they remember me"*

### Aesthetic Direction

**Primary direction: Airbnb-leaning warm discovery, with multi-vertical tonal sensitivity.**

The dominant feel is **photo-led, warm, aspirational** — closer to Airbnb than to Aesop. The venue's photography is the protagonist. Search and discovery are heroic. Cards breathe. The user should *want* to tap.

Inherited references (in order of weight):
1. **Airbnb** — photo-as-hero cards, search as the front door, warm aspirational tone, mass-friendly without losing taste
2. **Resy / Tock** — editorial typographic confidence in venue detail screens, cool restraint in dense content
3. **ClassPass / Mindbody** — efficient time-slot picking, calendar density done well
4. **Aesop / Kinfolk / Apartamento** — typographic personality, generous whitespace, restrained palette as a *background* taste filter, not a primary mode

**Tonal variation by vertical** — the master design system stays consistent, but accent color and iconography shift subtly to match the vertical's emotional register. Three sub-modes:

| Vertical group | Accent register | Photography mood |
|---|---|---|
| **Wellness & beauty** (spa, hair, nails, fitness) | Warm — terracotta `#C7834A`, ochre, dusty rose | Golden-hour, soft skin, hands, fabric, textured walls |
| **Healthcare** (dental, medical, aesthetics, therapy) | Calm — sage `#1F4D3A` deepened toward muted teal, off-white surfaces | Clean but human — hands, eyes, soft natural light, never sterile stock |
| **Retail by appointment** (jewelry, ateliers, custom) | Refined — deep ink `#1E1C18`, thin gold hairlines, more editorial weight | Object-focused, tactile, near-macro |

The color tokens in `src/theme/colors.ts` already lean this way. Future work should **extend** that palette into vertical-aware accents, not replace it.

**Theme**: Light mode only (locked via `app.json` `userInterfaceStyle: "light"`). Do not introduce dark mode without an explicit go-ahead.

**Surface palette baseline** (already in `src/theme/colors.ts`):
- Background `#F8F6F1` (warm bone, never pure white)
- Surface `#FFFFFF` (used for cards on bone)
- Surface muted `#EFEAE1` (warm beige for chips, secondary)
- Text `#1E1C18` (warm near-black, never `#000`)
- Muted text `#6F6A60` (warm taupe)
- Border `#DED7CA` (hairline taupe)
- Primary `#1F4D3A` (deep forest)
- Accent `#C7834A` (terracotta)
- Danger `#A33A2F` (warm brick)

**Typography**: System fonts are placeholder. Production should pair:
- A **display** with quiet personality (consider *Tiempos Headline*, *Söhne Breit*, *GT Alpina*, or a free alt: *Fraunces* / *Newsreader*) — used for screen titles, venue names, hero numbers
- A **text/UI** sans with high legibility at small sizes (consider *Söhne*, *GT America*, or free: *Inter Display* used sparingly, *Geist*, *General Sans*) — used for body, labels, buttons

Do **not** ship with system fonts in production — the app will read "templated" instantly.

### Anti-references — explicitly avoid

The user has explicitly rejected all four of these. Treat as red lines:

1. **Generic SaaS / AI look** — purple-blue gradients, glass cards, neon accents on dark, identical rounded-rect grids, "hero metric" templates, gradient text
2. **Corporate banking cold** — institutional blue, no personality, all 8px borders and white cards, sans-serif fortress
3. **Mexican hipster cliché** — cactus iconography, rosa mexicano + amarillo, talavera patterns as decoration, "artesanal" hand-drawn type, anything that reads tourist-facing
4. **Loud fitness gym** — black + neon, all-caps industrial type, sweating-people stock, exclamation marks, motivational copy

If a design choice would fit comfortably on any of those four reference boards, reject it.

### Design Principles

These five principles override individual taste calls:

1. **Photo of the venue earns the screen.** Every venue card, detail screen, and search result is photo-led. Photography is non-decorative — it's the trust signal. Reserve typography hierarchy for moments without photography (lists, settings, confirmation states).

2. **Warm trust over cold precision.** Even for medical and financial moments (booking a dentist, paying a deposit), choose warm clarity over clinical sterility. A hairline rule on warm bone is more trustworthy here than a hard shadow on pure white.

3. **Vertical-aware, not vertical-fragmented.** The system is one. Vertical accents (sage for healthcare, terracotta for wellness, ink for retail) are subtle filters applied to the same components — never a different component library, never a different layout grammar. A user moving from booking a haircut to booking a dentist should feel *oriented*, not transferred to a different app.

4. **Discovery-led copy, not transactional.** Microcopy leans toward "Encuentra tu próximo espacio", "Reserva con confianza", "Vuelve cuando quieras" — never "Manage bookings", "Submit reservation", "Process payment". Spanish first, mexicano natural, never robotic.

5. **Density is earned, not assumed.** Booking calendars and time-slot grids must be efficient (ClassPass-tight) when the user has committed to a slot decision. But discovery screens (home, search, venue detail) must breathe (Airbnb-loose). Switch modes deliberately by screen function — never average them.
